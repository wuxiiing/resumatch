# ResuMatch Project Workflow

## 角色分工

项目负责人 / 监理：麦桐。

- 确认需求边界。
- 提供 API Key、测试简历、测试 JD。
- 判断产品体验是否符合目标。
- 决定是否进入下一阶段。

Codex 总控窗口：贴身秘书和项目总负责人。

- 不做业务代码开发。
- 负责项目规划、任务拆分、风险提醒、验收标准和进度记录。
- 给其他执行窗口生成明确任务卡。
- 根据执行窗口的回传结果做验收判断。
- 有不确定点时先问项目负责人。

执行窗口：施工窗口。

- 一个窗口只执行一个明确任务。
- 只按任务卡工作，不自行扩展范围。
- 完成后回传改动文件、验证结果、问题和下一步建议。

## 工作规则

1. 本核心窗口是项目总控室，不直接写业务代码。
2. 需要开发时，由核心窗口生成任务卡，复制到新的执行窗口。
3. 执行窗口完成后，把结果复制回核心窗口验收。
4. 未经确认，不加入登录、付费、多语言、PDF 导出、一键修改等功能。
5. 所有 API Key 只放环境变量，不写进前端代码，不提交到 Git。
6. 每个阶段先跑通主链路，再补边界和体验。
7. 每次完成一个稳定阶段后再考虑 Git 提交。
