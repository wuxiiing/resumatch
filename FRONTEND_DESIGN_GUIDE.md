# ResuMatch Frontend Design Guide

> 本文件是前端设计执行依据，来源于 `ui_ux_handover_resumatch.md` 和项目负责人提供的结果页参考图。
> 前端施工窗口在做页面骨架、样式和交互时，优先遵守本文件；不得借此扩展第一版功能范围。

## 设计目标

ResuMatch 第一版前端应呈现为一个专业、清爽、可信的中文简历/JD 匹配分析工具。

核心观感接近参考图：

- 白色主背景。
- 极细边框。
- 圆润但克制的卡片。
- 左侧历史记录栏。
- 右侧大面积报告工作区。
- 青蓝色作为品牌强调。
- 绿色、黄色、灰色、红色承担报告语义。
- 信息密度接近工具型报告页，不做营销落地页。

## 风格关键词

- 极简
- 专业
- 可信
- 工具感
- 有温度
- 像“简历体检报告 / 查重报告”
- 不像后台管理系统
- 不像 AI 营销官网
- 不像游戏或科幻界面

## 第一版设计边界

必须避免：

- 不做登录注册入口。
- 不做付费入口。
- 不做多语言切换。
- 不做 PDF 导出入口。
- 不做网页内直接编辑简历。
- 不做一键修改、一键润色、一键生成简历。
- 不做雷达图、复杂技能图谱。
- 不做大面积渐变背景。
- 不做复杂炫酷动效。
- 不做明日方舟、游戏、科幻相关视觉。
- 不做传统后台管理系统的厚重导航风格。

## 品牌与 Logo

品牌名：`ResuMatch`

Wordmark 建议：

- `Resu` 使用深色主文字。
- `Match` 使用品牌青蓝。
- 下方可使用小字副标题：`RESUME · JD ANALYSIS`。
- 字体使用系统无衬线字体，保持轻盈。
- 第一版只使用静态 Logo，不做开场动画。

Logo 在参考图中的气质：

- 左上角显示品牌。
- 品牌下方有小号英文说明。
- 不需要复杂图形标。
- 不需要拟物化图标。

## 颜色系统

| 用途 | 建议颜色 | 使用说明 |
| --- | --- | --- |
| 品牌主色 | `#00B4CC` | Logo、重点按钮、选中边框、关键分数强调 |
| 深青蓝 | `#0891B2` | 分数渐变、hover、强调文本 |
| 匹配绿色 | `#22C55E` | 与 JD 高度相关的段落和标签 |
| 匹配浅绿背景 | `#ECFDF5` | 绿色标注段落背景 |
| 可优化黄色 | `#EAB308` | 建议优化的段落和标签 |
| 可优化浅黄背景 | `#FFFBEB` | 黄色建议展开区域背景 |
| 无关灰色 | `#9CA3AF` | 与目标岗位关联度较低 |
| 灰色浅背景 | `#F8FAFC` | 灰色标注段落背景、次级容器 |
| 缺失红色 | `#EF4444` | 缺失关键词、错误、超限 |
| 缺失浅红背景 | `#FEF2F2` | 缺失关键词标签背景 |
| 主文字 | `#0F172A` | 标题、正文 |
| 次级文字 | `#64748B` | 说明、时间、辅助信息 |
| 边框 | `#E2E8F0` | 卡片、输入框、分割线 |
| 页面背景 | `#FFFFFF` | 主背景 |

颜色使用原则：

- 页面整体以白色和浅灰为主。
- 青蓝色只用于品牌和重点行动，不铺满页面。
- 报告语义颜色必须稳定：绿色=匹配，黄色=可优化，灰色=弱相关，红色=缺失或错误。
- 不使用大面积紫色、黑色、游戏化霓虹色。

## 字体与质感

字体：

- 使用系统字体栈。
- 中文优先使用系统默认中文无衬线。
- 英文和数字保持清晰、轻盈。

文字层级：

- 品牌 Logo：清晰但不过大。
- 报告分数：最大视觉重点。
- 卡片标题：中等字号，字重 600 左右。
- 正文：适合长时间阅读。
- 说明文字：使用次级文字色。

质感：

- 边框 1px 或更细。
- 圆角建议 10px 到 14px。
- 阴影很轻，不能有厚重悬浮感。
- 卡片留白适中，不要做成营销卡片堆叠。

## 首页布局

首页第一版保持简单，和参考图风格一致，但不做报告页那样复杂。

建议结构：

```text
[顶部/居中 Logo：ResuMatch]
[一句话说明：上传简历 + 粘贴岗位 JD，生成匹配分析报告]

[主输入面板]
  [简历上传区]
    支持 .pdf / .docx / .xlsx
    显示上传状态占位

  [JD 输入区]
    textarea
    右下角字数计数
    1000 字限制提示

  [开始分析按钮]
    未满足条件时禁用

[底部说明]
  每个 IP 每日最多分析 5 次
  本报告由 AI 生成，仅供参考
```

首页原则：

- 不做营销型 hero。
- 不做左右复杂宣传区。
- 不做功能介绍长页面。
- 不做注册登录入口。
- 上传和 JD 输入是首屏核心。
- UI 风格要和报告页一致：白底、细边框、青蓝强调。

## 报告页布局

报告页应尽量接近参考图，是第一版最重要的视觉目标。

整体结构：

```text
[左侧栏：历史记录]
[右侧主区域：分析报告]
```

### 左侧栏

宽度建议：

- 桌面端约 260px 到 320px。
- 参考图中左栏约占 20%。

内容：

```text
[Logo]
[搜索历史记录输入框]

[历史记录列表]
  公司名 / 岗位名
  时间
  匹配度百分比

[清空历史记录按钮]
```

视觉要求：

- 当前选中记录使用青蓝边框和浅背景。
- 每条记录是白色圆角卡片。
- 分数按语义着色：高分绿色，中等橙黄，低分红色。
- 历史记录第一版可以是本地占位或 localStorage，不能引入登录系统。

### 右侧顶部栏

参考图右上角有：

```text
[+ 新建分析] [用户圆形占位图标]
```

第一版处理：

- 可以保留“新建分析”按钮，用于返回首页或清空当前分析状态。
- 用户头像只能作为静态占位或不显示，不能引入登录功能。
- 如果担心误解为账号系统，优先不显示头像。

### 匹配度评分卡

这是报告页第一视觉重点。

结构：

```text
[匹配度评分]
[大号分数 87] [/100]
[一句整体评价]

[已匹配关键词标签]
[缺失关键词标签]
```

视觉要求：

- 大号分数使用青蓝色，可做轻微青蓝渐变。
- `/100` 使用灰色。
- 已匹配关键词使用浅绿标签。
- 缺失关键词使用浅红标签。
- 卡片为白底、细边框、圆角。

### 图例说明

放在报告主体上方：

```text
● 绿色：与 JD 高度相关，建议保留
● 黄色：内容有价值，建议优化表达
● 灰色：与目标岗位关联度较低
```

要求：

- 图例必须清楚，因为报告颜色依赖它理解。
- 不要做成复杂说明区。

### 简历分析详情区

左侧主内容，参考图中占报告主体大部分宽度。

结构：

```text
[简历分析详情]
[工作经历 / 项目经历等分组标题]

[绿色段落卡片]
  原文
  标签：与 JD 匹配

[黄色段落卡片]
  原文
  标签：可优化
  展开建议：
    问题说明
    修改建议
    示例文本

[灰色段落卡片]
  原文
  标签：可考虑精简
```

视觉要求：

- 绿色段落使用浅绿背景和绿色边框。
- 黄色段落使用浅黄背景和黄色边框。
- 灰色段落使用浅灰背景和灰色边框。
- 黄色建议展开区域要像参考图一样直接嵌在段落下方。
- 修改建议示例使用浅灰小块承载。
- 第一版不提供网页内编辑能力。

### 右侧摘要栏

参考图右侧有三块：

```text
[整体评语]
[优化建议汇总]
[导出报告]
```

整体评语：

- 2 到 4 句中文。
- 口吻温和、教练式、客观。
- 不替用户重写整份简历。

优化建议汇总：

- 按类别展示数量，例如：
  - 技能层面
  - 项目经验
  - 成果量化
- 数字可以使用浅色圆形 badge。

导出报告：

- 展示 Word 图标或简洁文件图标。
- 主按钮文案：`下载报告` 或 `下载 Word 报告`。
- 辅助说明：`包含标注说明和修改建议清单`。
- 不出现 PDF 导出按钮。

## 移动端布局

移动端原则：

- 左侧历史记录收起到顶部，或放到报告下方。
- 报告主体变为单列。
- 分数卡保持顶部优先。
- 关键词标签允许换行。
- 简历段落卡片要保持可读，不要压缩成表格。
- 右侧摘要栏移动到简历详情下方。

移动端顺序建议：

```text
Logo / 顶部操作
匹配度评分
关键词
图例
简历分析详情
整体评语
优化建议汇总
导出报告
历史记录
```

## 交互原则

首页：

- 上传区支持点击和拖拽的视觉反馈。
- JD 输入显示字数计数。
- 超过 1000 字时计数变红，提交按钮禁用。
- 简历未上传或 JD 为空时，开始分析按钮禁用。

分析中：

- 按钮进入加载状态，禁止重复点击。
- 可显示文字状态：
  - 正在解析简历内容...
  - 提取岗位关键词...
  - 对比匹配度...
  - 生成分析报告...
- 第一版不需要复杂进度条。

报告页：

- 黄色可优化段落可展开/收起建议。
- 绿色和灰色段落可以有 hover 提示。
- 导出 Word 按钮只触发 Word 导出，不出现 PDF 选项。

## Word 导出界面要求

页面上的 Word 导出区域应说明导出内容：

- 分析报告摘要。
- 简历原文干净版。
- 优化建议清单。

明确不要在 Word 中做颜色标注。Word 文件应方便用户自行修改。

## 给前端施工窗口的执行要求

前端施工窗口在阶段 1 做骨架时：

- 可以先做静态占位数据。
- 必须先搭出接近参考图的信息结构。
- 不要接入真实 API。
- 不要实现真实上传解析。
- 不要实现真实 Word 导出。
- 不要创建登录入口。
- 不要把头像做成账号系统。
- 不要把首页做成营销落地页。

阶段 1 骨架验收重点：

- 首页能打开。
- `/result` 能打开。
- 页面风格已经接近白底、细边框、青蓝强调的参考图。
- 报告页布局接近参考图：左侧历史栏 + 右侧评分卡 + 详情区 + 右侧摘要卡片。
- 所有文案为中文。
- 没有范围外功能入口。

## 参考图关键特征清单

前端实现时应尽量贴近以下特征：

- 左上角 `ResuMatch` 品牌，其中 `Match` 为青蓝色。
- 左侧栏有历史搜索框和多条历史分析记录。
- 当前历史记录使用青蓝边框高亮。
- 主区域顶部是一张大评分卡。
- 评分 `87/100` 使用超大数字。
- 匹配关键词是浅绿色标签。
- 缺失关键词是浅红色标签。
- 报告主体上方有三色图例。
- 简历段落卡片按绿色、黄色、灰色分层。
- 黄色段落下方展开问题说明和修改建议。
- 右侧有整体评语、建议汇总、导出报告三张卡片。
- 底部有 AI 参考免责声明。
