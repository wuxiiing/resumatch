## 阶段（按顺序，每步做完都能用）

### 阶段 1 · 主线接真（做完它就是个能用的产品）
目标：用户喂自己的 JD＋简历＋意图 → 看到**真** agent 研判（不再是 mock）。
- **1a 输入页** `app/agent/page.tsx`（国风，client）：JD 文本框 ＋ 简历上传 ＋ 意图（目标方向 / 绝不接受）＋「岗位·公司」字段 ＋ 开始分析。复用 `/api/parse-resume` 把简历转文本。
- **1b 接分析**：提交 → `/api/agent-analyze`。**好的加载态**：按 ①解读→②证据→③判断→④策略 显示进度（慢，4 串行调用；顺便把 agent 架构秀出来，作品集加分）＋ 错误处理。
- **1c 结果页吃真数据**：`agent-result` 由 mock 改为读 sessionStorage 里的真输出（数据形状已对齐，渲染层基本不动）；转成 client 组件。
- 注：JD「类型」那格（硬门槛/加分）是节点 ① 的**深加工**（改 prompt/schema），Phase 1 先按现有输出渲染，「类型」作为 1.5 增强补。
- **验收**：浏览器端到端走通 —— 真 JD ＋ 真简历 → 真研判页。

### 阶段 2 · 军师对话（真多轮，不是空壳）
- **2a** `/api/agent-chat`：上下文 = 报告 ＋ 简历 ＋ JD ＋ 对话历史 → DeepSeek 多轮。先做**简单但真**（非占位），不强求一上来就 LangGraph human-in-the-loop。
- **2b** 结果页右栏对话面板（工作台右栏）＋ 输入框，接 2a。
- **验收**：追问能得到扎根于本报告的真回答。

### 阶段 3 · 简历改写＋导出（能行动）
- **3a** 建议改写卡（before→after，源于对话 / ④ 策略）。
- **3b** 网页里编辑简历工作副本（可改）。
- **3c** 导出：Word（复用 `/api/export-word`）＋ PDF（新增）。
- **验收**：采纳一条建议 → 导出 → 打开文件正常。

### 阶段 4 · 本地历史
- 浏览器本地存分析记录（复用 MVP 本地历史思路）＋ 列表 ＋ 重开。
- **验收**：两次分析都在列表、可重开。

## 铁律（贯穿）
不动 MVP；不做假占位；PII（真简历/JD）不提交；Windows dev server 用 bounded 方式；过程状态写进文件。
