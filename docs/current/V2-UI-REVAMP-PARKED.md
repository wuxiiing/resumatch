# 暂存 · 报告/网页 UI 改版方案（2026-06-26）

> 决策:**先把功能全部做完,再统一搞网页。** 此方案存档,功能齐了照做。
> 来源:用户对 `/agent-result` 首版(卡片式)的反馈。

## 一、报告格式（去卡片）
- 全部去掉描边卡片框,改「**竹节标题 + 发丝线 + 内容流下**」的论文/古籍分点式(和现报告后半段一致)。
- **保留竹节 bullet;不要汉字数字。**
- **章节大顺序不变。**
- **取舍、时限**:从判定框里拎出,做两个独立竹节小节,放判定正下方。
- 判定区精简到:节节高尺 +「档位 · 边界」(盖小朱印)+ 一句定性 + 匹配信号一行。

## 二、治"文字累"（加表 + 装饰，克制）
- 逐条排雷 / 牌面 / 潜台词 等「逐条」内容 → 改**国风细线表**(只发丝线、宋体、留白,**不是 SaaS 网格**),一眼能扫。
- 叙述性内容(核心洞察、应对策略)保留行文。
- 装饰:背景竹影再加一两枝(疏密错落);关键短语用**圈点/着重号**代替西式加粗;大节之间一道淡发丝分隔;小朱印。

## 三、右侧导航 + 多功能台
- 导航(左/右待定,用户倾向右):
  - **功能区**:岗位研判 · 军师对话 · 修改简历 · 职业规划
  - **历史研判**:麦桐 × 各公司,浏览器本地存,可重看
- 主区按功能切换,默认 = 当前研判报告。
- **诚实:只露已做好的功能,不放"即将推出"假占位**(用户明确这是面试减分)。
