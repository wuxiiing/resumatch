  2. **军师对话** — 需新建多轮 LangGraph（human-in-the-loop）,后端未建,前端先留位。
  3. **简历对照·改写·导出 PDF/Word** — 复用已有 Word 导出后端（`/api/export-word`）+ 加 PDF + 行内编辑,后端待建,前端先留位。

## 2. 视觉系统（国风 · 服务级）
- 母题：**竹**。寓意 步步高升 / 气节 / 韧（贴"求职"）。**核心纪律：服务型产品,靠结构+内容撑分量,别堆装饰、别做成设计作品。**
- 配色：暖宣纸黄底（~`#f2e8d3`,偏暖黄、**不偏绿**）+ 竹青 `#52724b`（功能色：链接 / 命中 / 章节 / 节节高）+ 墨字 `#23271f` + 印章红 `#9c3b30`（仅判定 / 避雷点缀）。
- **宣纸纹理**：SVG `feTurbulence` 程序生成（不贴图）。比 demo 左图**略重一档** + 加**长纤维**（各向异性 baseFrequency,横向纤维感）,仿真但不重。叠在暖底、不伤正文可读。
- 字：**宋体**（标题 / 章节 / 结论 / 引文）；**无衬线**（表格 / 数据 / 标签 / 密集正文）。
- 竹影：右上低透明度（~0.1）一枝淡竹,氛围,不抢内容。
- **节节高判断尺（签名元素，保留）= 岗位定位轴，不是匹配度！**：该绕开 < 跳板 < 目标 三段竹节升阶,当前档高亮 + "你在此",每档配一句义（该绕开=别浪费枪 / 跳板=过渡练级 / 目标=值得扎根）。**明确标注「岗位定位 · 不是打分」**,杜绝被当成分数。
  - **定位 vs 匹配 = 两个独立的轴,别混**：定位 = 对你目标值不值得追;匹配 = 经历贴不贴 JD。可背离（高匹配却该绕开 / 低匹配却是目标）。
  - **评分收口（决策 2026-06-25）**：新管线不产出 0-100 分。**定位（节节高）当头牌**,**匹配用「逐条命中」表达**（硬性要求命中 X/Y，从『逐条排雷』派生,诚实可追溯）,不再设不透明分数。纯前端派生,不动后端。
- 随风飘动：**仅**封面 / 加载轻摇曳（CSS/GSAP）,正文不动。
- 标题 = 「岗位 + 公司」；判定结论放报告**内部**醒目区,**不**做总起句标题。

## 3. 报告文档结构（这一版要做的）
1. 页眉：求职军师·研判 / 标题（岗位+公司）/ 分析对象·日期·基于简历与意图。
2. **判定区（醒目）**：节节高尺 + 跳板·边界 + 定性 + 取舍 + 时限。
3. **① JD 真身解读**：真身 / 要什么人 + 逐句潜台词（更细）+ 信息缺口。
4. **② 你的牌面**：核心牌面 + 证据（原文 → 证明）。
5. **③ 逐条排雷**：要求 | 命中 | 研判 表 + 避雷 + 条件分叉。
6. **④ 应对策略**：扬长 / 避短 + 面试要点 + 谈薪。
7. 页脚：免责（不编造、可追溯）。

- **破单调**：不同内容用不同呈现——关键洞察（如"PM = 需求翻译官"）拎成醒目「金句 / 洞察块」；判定用节节高；覆盖度可加小可视化。**不是加装饰。**
- **JD 更细**：每条需求拆 `硬门槛/加分项 → 潜台词 → 你缺什么 → 怎么补`。**现在先用模拟内容展示深度**；真深度要等接 LLM 时扩节点 ① 的 prompt/schema（届时做,不碰旧 MVP analyze 链）。

## 4. 数据来源（重要，别混）
- 现在 mockup 里的字 = **手写模拟**（照真实「丁香园」验收编的）。真内容 = LLM 4 节点输出。
- "够不够详尽"= 两个旋钮：节点 ① 输出多少（prompt/schema）× 前端怎么铺。**深加工 = 接后端那步,不是现在。**
