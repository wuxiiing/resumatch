# ResuMatch 正式版上线前检查清单

> 本文件记录正式版上线前必须收口、确认或后续再考虑的事项。
> 这里不是施工任务池，不代表马上要做；总控室应在合适阶段提醒项目负责人，由项目负责人决定是否生成任务卡。

## 使用规则

1. 任何“开发验收阶段可以存在，但正式版不应明显出现”的内容，都记录到这里。
2. 任何“以后再考虑”的体验、配置、部署、风险项，也记录到这里。
3. 总控室在进入 checkpoint、部署、视觉验收或阶段切换前，应主动检查本文件。
4. 本文件只做提醒和决策依据，不替代 `TASKS.md` 的阶段任务，也不替代 `PROJECT_SCOPE.md` 的产品范围。
5. 如果项目负责人决定执行其中某项，总控室再生成单独任务卡。

## 开发验收可见，正式版需收口

- [ ] 首页 opening animation 的“重播”按钮：开发验收阶段可见；正式版应隐藏，或放在非常不明显的位置。
- [ ] 所有施工提示、阶段提示、占位说明、mock 说明：正式版不得出现给用户看。
- [ ] 历史记录里的 mock 数据：正式版上线前需要确认是保留为本地示例、改成本地真实历史，还是隐藏空状态。
- [ ] 报告页 mock 分析内容：接入真实分析链路后必须替换，避免用户误以为是自己的报告。
- [ ] 任何调试按钮、临时入口、测试端口说明：正式版不得出现在页面 UI 中。

## 首页正式版检查

- [ ] 首页首屏应保持紧凑，一眼能看到 Logo、上传简历、目标岗位描述、开始分析按钮和轻量提醒。
- [ ] 上传区应保持附件式紧凑文件条，不恢复成大面积拖拽卡片。
- [ ] 文案不要只写 `JD`，应使用“目标岗位描述 / 招聘要求 / 岗位职责和任职要求”等普通用户能理解的表达。
- [ ] 底部提醒保持轻量，不做成大卡片。
- [ ] opening animation 不得遮挡用户操作太久，不做全屏长开屏，不做残影，不斜飞。
- [ ] opening animation 必须支持 `prefers-reduced-motion`。
- [ ] 移动端首页不能横向溢出，文字不能重叠，按钮不能不可点。

## 报告页正式版检查

- [ ] 报告页顶部应清楚展示匹配度、关键词和整体判断。
- [ ] 右侧摘要栏第一块保持“岗位方向”，不要重复顶部整体评价。
- [ ] 绿色、黄色、灰色标注语义必须稳定清楚。
- [ ] 黄色优化片段必须能看到问题说明和建议示例。
- [ ] 报告页只读，不提供网页内直接编辑简历能力。
- [ ] 导出区域只能提供 Word 导出，不出现 PDF 导出按钮。
- [ ] 历史记录入口不得暗示登录或账号系统。
- [ ] 移动端报告页至少基础可读，不横向撑爆，不出现明显布局破裂。

## 第一版禁止项复查

上线前必须确认页面和代码中没有以下入口或暗示：

- [ ] 登录 / 注册。
- [ ] 付费 / 会员 / 订阅。
- [ ] PDF 导出。
- [ ] 网页内直接编辑简历。
- [ ] 多语言切换。
- [ ] 一键修改 / 一键润色 / 一键生成简历。
- [ ] 未批准的营销页、复杂功能介绍页或账号体系。

## 技术上线检查

- [ ] `npm run lint` 通过。
- [ ] `npm run build` 通过。
- [ ] 本地 dev server 可访问 `/` 和 `/result`。
- [ ] 生产预览或 Vercel 预览可访问 `/` 和 `/result`。
- [ ] 如果 `next start` 出现本地产物错误，必须单独复查，不能直接忽略。
- [ ] 不提交 `.env.local` 或真实 API Key。
- [ ] secret/token/API key 扫描无真实密钥。
- [ ] 依赖文件变更必须有明确原因和验收记录。
- [ ] Git 工作区在 checkpoint 前应干净或变更范围清楚。

## 后端和业务链路上线检查

- [ ] `.docx` 简历文本提取稳定。
- [ ] `.xlsx` 简历文本提取稳定。
- [ ] 文字型 `.pdf` 简历文本提取稳定。
- [ ] 百度 OCR 兜底是否接入第一版，需要项目负责人确认。
- [ ] 简历文本超过 3000 字时拒绝提交，并给出清楚提示。
- [ ] 岗位描述超过 1000 字时阻止提交。
- [ ] DeepSeek `deepseek-chat` 只在服务端调用。
- [ ] DeepSeek 返回 JSON 必须做结构校验。
- [ ] JSON 解析失败时重试一次，仍失败时返回清楚错误。
- [ ] 报告页渲染必须基于结构化结果，不依赖纯文本拼接。
- [ ] Word 导出为干净文本和建议附录，不加入颜色标注。
- [ ] IP 限流方案需要确认：内存 MVP 还是 Vercel KV。

## 部署前待确认

- [ ] 是否已有 DeepSeek API Key。
- [ ] 是否已有百度 OCR API Key。
- [ ] Vercel 环境变量是否配置完成。
- [ ] 每 IP 每日 5 次、全站每日 500 次的限制是否已生效。
- [ ] 部署后用真实中文简历和真实岗位描述做完整验收。
- [ ] 正式访问地址是否需要写入 README 或项目负责人交接文档。

## 以后再考虑，不影响第一版

- [ ] opening animation 正式版是否完全隐藏重播按钮。
- [ ] 正式版前再评估 opening animation 是否从“动画 Logo 对齐并交接给主页真实 Logo”升级为“动画 Logo 直接成为主页 Logo”；只有在首页布局、移动端表现、重播策略和 reduced-motion 策略稳定后再考虑。
- [ ] 是否增加更细腻的移动端报告阅读模式。
- [ ] 是否增加更完整的历史记录管理。
- [ ] 是否增加更友好的错误恢复和重试提示。
- [ ] 是否增加更精细的导出 Word 样式。
- [ ] 是否在第二版考虑账号、付费、多语言或 PDF 导出；第一版不做。
