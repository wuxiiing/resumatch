# ResuMatch Project Scope

> 按交接文件实现，有不确定的地方先问项目负责人，不要自行扩展功能范围。

## 项目定位

ResuMatch 是一个中文简历与岗位 JD 匹配分析系统。用户上传简历并粘贴目标岗位 JD 后，系统分析匹配度，生成类似查重报告的可视化标注报告，并导出干净的 Word 报告，供用户参考后自行修改简历。

核心原则：系统只做分析、标注和建议，不替用户重写简历，不制造一键改写体验，尽量保留用户自己的表达风格。

## 第一版必须做

- 支持上传 `.pdf`、`.docx`、`.xlsx` 简历文件。
- 从简历文件中提取纯文本。
- 简历文本超过 3000 字时提示用户精简，并拒绝提交分析。
- 支持粘贴中文岗位 JD。
- JD 超过 1000 字时阻止提交。
- 调用 DeepSeek API，使用 `deepseek-chat` 模型返回结构化 JSON。
- 展示匹配度分数、已匹配关键词、缺失关键词、优化建议数量。
- 在报告页按 `relevant`、`optimize`、`irrelevant` 标注简历文本。
- `optimize` 片段展示问题说明和修改建议示例。
- 支持导出 Word 报告。
- Word 报告保持干净文本，不做颜色标注。
- 基于 IP 做次数限制：每 IP 每天 5 次，全站每天 500 次。
- 全中文界面、全中文输出。

## 第一版明确不做

- 不做用户注册或登录。
- 不做 PDF 导出。
- 不做网页内直接编辑简历。
- 不做多语言。
- 不做付费系统。
- 不做一键修改、一键润色、一键生成简历。
- 不自行扩展交接文件以外的功能。

## 技术边界

- 前端框架：Next.js App Router。
- 样式：Tailwind CSS，先使用基础样式。
- 后端：Next.js API Route / Vercel Functions。
- AI：DeepSeek API。
- PDF 解析：pdf.js，失败后走百度 OCR 兜底。
- Word 解析：mammoth.js。
- Excel 解析：xlsx.js。
- Word 导出：docx.js。
- 部署：Vercel。

## 验收口径

一个阶段只有在满足以下条件后才算完成：

- 功能符合交接文件范围。
- 没有加入未批准的新功能。
- 能说明改动了哪些文件。
- 能说明实际运行过哪些验证。
- 如有未验证项，必须明确列出。
